Paquete: biomaRt

El paquete biomaRt ‚Äúproporciona una interfaz a una creciente colecci√≥n de bases de datos que implementan la suite de software BioMart (http://www.biomart.org). El paquete permite la recuperaci√≥n de grandes cantidades de datos de manera uniforme sin necesidad de conocer los esquemas de las bases de datos subyacentes o escribir consultas SQL complejas‚Äù. Por lo tanto toda la informaci√≥n disponible en la versi√≥n en l√≠nea de de Biomart, utilizando R.

Aqu√≠, utilizaremos el mismo archivo de entrada utilizado en la versi√≥n en l√≠nea de Biomart.

#El siguiente comando se puede utilizar para instalar el paquete
#(s√≥lo es necesario eliminar el car√°cter de comentario #)

Tras cargar el archivo de entrada y cargar el paquete, es posible iniciar la b√∫squeda de genes dentro de nuestro intervalo seleccionado.

  1. El primer paso es elegir la base de datos y el organismo para elegir los genes
#El siguiente comando muestra las bases de datos que est√°n disponibles para recuperar informaci√≥n
#listMarts()

#Elegiremos ENSEMBL_MART_ENSEMBL, que corresponde a Ensembl Genes 91
#Es necesario crear un objeto del tipo "mart" para cargar la informaci√≥n de la base de datos.

#Esto se realiza con el siguiente comando:
mart <- useMart("ENSEMBL_MART_ENSEMBL")

#Una vez creado el objeto mart mediante la funci√≥n useMart(), necesitamos seleccionar
#el organismo. El siguiente comando se puede utilizar para identificar el correspondiente
#argumento a cada organismo.

#listDatasets(mart)

#La funci√≥n useDataset() puede utilizarse para insertar el organismo
#informaci√≥n en el objeto mart.
mart <- useDataset("hsapiens_gene_ensembl", mart)
  1. Ahora es el momento de definir los filtros y atributos que se recuperar√°n de la base de datos.
#Definir el filtro que se utilizar√° para recuperar la informaci√≥n
filter<-"external_gene_name"

#Crear un vector con los DEG
value<-deg.list$Gene

#Definir la informaci√≥n a recuperar de la base de datos 
attributes <- c("external_gene_name","go_id","name_1006",
                "go_linkage_type", "namespace_1003")
  1. Ahora, tras definir los filtros y los atributos, es el momento de recuperar la informaci√≥n
#En este punto, necesitamos informar los objetos creados en los pasos anteriores como argumentos
#de la funci√≥n getBM()
all.genes <- getBM(attributes=attributes, filters=filter, values=value, mart=mart)

#Eliminaci√≥n de genes sin anotaci√≥n
all.genes<-all.genes[-which(all.genes$namespace_1003==""),]

#Comprobar la salidastr(all.genes)
head(all.genes)
#Guardar la salida
write.table(all.genes, file="output_biomart_DEG.txt",
            row.names=F, sep="\t", quote=F)

Paquete gprofiler2

Del mismo modo, podemos realizar el mismo an√°lisis para las rutas metab√≥licas. En este caso, utilizaremos el paquete gprofiler2 para realizar la anotaci√≥n utilizando v√≠as KEGG y Reactome.

El paquete gprofiler2 es un conjunto de herramientas para el an√°lisis y la visualizaci√≥n del enriquecimiento funcional, la conversi√≥n de identificadores gen/prote√≠na/SNP y el mapeo de genes ort√≥logos entre especies a trav√©s de ‚Äòg:Profiler‚Äô.

library(gprofiler2)

#Anotaci√≥n de v√≠as KEGG y Reactome mediante Gene symbols
res.anno <- gost(query = deg.list$Gene, 
                   organism = "hsapiens", exclude_iea = FALSE, 
                   measure_underrepresentation = FALSE, evcodes = TRUE, 
                   user_threshold = 1, correction_method = "fdr", 
                   domain_scope = "annotated",
                   numeric_ns = "", sources = c("KEGG", "REAC"))

#Filtrar la tabla de resultados
res.anno<-res.anno$result

head(res.anno)
